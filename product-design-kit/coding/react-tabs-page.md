# React 标签页开发方法

> 当一个领域需要多个子页面共享同一层导航时，再使用标签页或路由容器模式。

## 推荐结构

```text
src/views/<Domain>/
  <Domain>Tabs.tsx
  <Domain>-A.tsx
  <Domain>-B.tsx
src/router/index.tsx
```

## 实现原则

1. 标签页容器只负责路由切换和公共布局，不承载具体业务逻辑。
2. 每个子页面仍然是独立页面文件，方便单独设计、单独测试和后续拆分。
3. 优先使用路由 + `Outlet`，不要把所有标签页内容堆在一个文件里。
4. 如果子页面之间共享概念、规则或状态边界，在进入实现前先把领域结构和页面关系补进 `design.md`。

## 子页面职责

- 子页面自己维护数据、状态、事件和局部弹窗。
- 容器页面只维护标签配置、默认落点和导航结构。
- 子页视觉层级、组件气质和密度默认继承 `project/ui-brand.md`。
- 菜单、路由和设计文档要一起更新。

## 何时使用

- 一个领域包含 2 个及以上紧密相关的页面
- 这些页面共享上层导航或共同入口
- 用户会在多个子页之间高频切换

## 何时不要使用

- 只有一个简单列表页
- 页面之间几乎没有共享上下文
- 只是为了把大文件拆开，而不是为了建立清晰导航

## 最小自查

- 默认路由是否正确落到第一个子页
- 标签切换后地址是否正确
- 每个子页能否独立打开和渲染
- 子页文案、状态和交互是否与设计一致
