## 分层思路

- 本领域分层维度:**按模块(控制台 / 会话中心 / 发送合规 / 辅助)+ 按关键规则(状态机跃迁、指派回滚、失败分类、违禁词方向)组合**。
- 选择理由:核心复杂度集中在"状态机 × 指派 × 发送结果"耦合规则上,纯按页面分层会漏掉跨页规则(如隐式指派失败回滚),故叠加规则维度补强。

## 优先级

| 优先级 | 划分标准 |
| --- | --- |
| P0 | 核心业务流程、数据准确性、权限安全(状态机、指派、发送、违禁词、主动发起落库) |
| P1 | 分支流程、关键校验、状态流转(转接、结束、搜索、号切换) |
| P2 | 边界、异常恢复、兼容性(标记、置顶、通知、撤回引导) |

## 回归范围

- 进回归集:所有 P0 用例、状态机 × 指派 × 发送结果耦合用例、主动发起落库用例、跨页(玩家中心 → 工作台)联动用例。
- 不进回归集:纯文案 / 纯样式、一次性扫码登录初始化。
- 本领域特殊回归点:**隐式指派失败回滚 vs 主动发起不回滚**两条规则不能互相污染;会话 ID 跨轮次不变。

## 风险

| 风险项 | 风险描述 | 影响 | 对策 |
| --- | --- | --- | --- |
| RPA Mock 与真实链路差异 | 发送结果判定时延 / 失败类型在真实链路可能不同 | 中 | 失败分类用枚举建模,接真实链路只换数据源 |
| 状态机回滚规则复杂 | 隐式指派失败回滚 / 主动发起不回滚易混淆 | 高 | 用例显式区分两路径;回归集常驻 |
| 长连推送稳定性 | 断连期间状态 / 未读不同步 | 中 | 重连横幅 + 增量补齐用例覆盖 |

## 退出标准

- 所有 P0 用例已执行并通过;状态机 / 指派 / 发送三类耦合用例全绿。
- 高优先级问题已关闭或记录风险。
- 回归集已整理;`strategy.md` 与 `cases.md` 可互相回溯。
