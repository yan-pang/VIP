# Ops-admin 决策记录

> 本领域的决策、迭代历史与阶段性实现口径（倒序，新在上）。`design.md` 只保留当前稳定产品设计；实现状态统一由 `project/TASKS.md` 维护。

## 2026-07-22 — 生命周期、风控基准与服务边界（覆盖同日及更早冲突口径）

- 企微接入配置不再提供物理删除，只允许启用 / 停用。停用立即停止自动化并移出新接待范围，但历史会话、配置、授权引用与资源绑定保留；重新启用后仍需显式恢复机器人，不自动启动。
- 支持多个企业微信企业主体；账号身份带稳定 `corpId`，玩家外部联系人身份以 `corpId + external_userid` 唯一。`accountId` 仍是 ChatFlow 内部账号键，不能替代外部身份键。
- 合规出口基准由阿里云资源同步的“分配公网 IP”提供，不允许把当前探测 IP 自写为基准。运行前和运行中比较当前出口与分配基准；未检测或漂移时告警并硬拦截发送 / 上线 / 接管。
- 所有运维写操作必须在服务层校验已认证身份、角色、数据范围和目标版本；成功、拒绝、失败都写审计，不使用硬编码操作人。配置采用乐观锁，冲突要求刷新后重试。
- 长生命周期执行单元统一叫“机器人常驻任务”；工作台每次提交是“消息发送批次 / 执行”，以请求幂等键关联逐条结果。

## 2026-07-22 — V1 Mock 当时验收快照（后续由一致性整改覆盖）

- 当时验收确认账号状态与风控状态分离、IP 合规结果归属绑定关系；常驻执行单元后来统一命名为“机器人常驻任务”。
- 风险告警默认聚焦待处理与已确认事件；配置同步故障转内部技术监控；操作审计页面只展示发生时间、操作人、动作和结果。
- 本领域停止继续扩展 Mock，后续变化仅由外部接口、试运行规模或风控 / 合规参数对齐触发。

## 2026-07-22 — 常驻任务范围收敛（原文称“RPA 任务”）

- 当前机器人常驻任务只有“常驻发送任务”，每个机器人最多一条；“消息接收与归档”不属于该常驻任务。工作台每次提交另称“消息发送批次”。
- 资源列表的任务摘要直接展示“机器人常驻发送任务 + 当前状态”，任务详情最多显示一行，并保留状态、原因和最近变更时间。

## 2026-07-22 — 风险告警边界与 IP 检测口径

- 单号单桌面由企微接入 / 改绑保存时的唯一约束保证；冲突直接拒绝并写操作审计，不生成风险告警。
- 配置事件投递与下游刷新失败属于平台内部可靠性监控，不在运营风险告警中展示；敏感操作执行前仍向权威服务校验最新配置与权限。
- 云电脑侧检测当前出口 IP，风控服务将其与所绑定企微号的合规出口要求比较；校验结果属于企微号的风控状态，不属于云电脑自身状态，因此云电脑 & RPA 列表不展示 IP 校验列。资源启动 / 恢复、机器人上线和人工接管前执行校验，运行中定时复检；漂移时阻止上线和接管并生成告警。
- 控制台和企微号配置统一展示账号状态“在线 / 离线 / 封禁”；风控状态只展示“正常 / IP 待校验 / IP 异常 / 频率受限”，账号已封禁时显示“—”，不再重复展示封禁。
- 操作审计列表只展示发生时间、操作人、动作和结果，删除目标、摘要和关联 ID 三列；后台仍保留目标与关联信息用于权限裁剪和故障追踪。

- 补齐云电脑与 RPA 资源页、风控告警生命周期、操作审计、应急回退和配置传播规则。真实接入的接口与可靠性契约沉淀到项目级研发设计，不在领域产品设计中维护 API 细节。

## 2026-07-22 — 风控策略与审计入口

- 运营管理新增明确的 `/ops-admin/operational-events` 入口，页面内按“风险告警 / 风控策略 / 操作审计”三个标签组织，不再只把风控和审计写成后台能力。
- 系统管理员可确认 / 解决告警，并在服务端硬上限内调整每企微号发送频率与连续失败阈值；调整原因必填，保存后策略版本递增并写入审计。硬上限不可在页面修改或关闭。
- 运营主管的顶栏“运营管理”直接进入该页面，只读自身授权企微号范围；客服无入口。配置、资源与违禁词页面仍仅系统管理员可访问。

## 2026-07-22 — 当前 Mock 到生产目标的迁移口径

| 数据 / 能力 | 当前 V1 Mock | 生产目标 |
| --- | --- | --- |
| 企微号与游戏归属 | `opsAdminMock` 共享 Store，并同步到既有号清单 | ops-admin 配置服务是唯一写入源，按配置版本发布变更事件 |
| 兼容游戏映射 | 保存时同步 `gameCatalogMock.mockWechatGameMap` | 移除前端镜像，permission 直接消费版本化配置或服务端授权结果 |
| 云电脑与 RPA 资源 | 本地资源种子、状态切换和任务摘要 | 从阿里云 / RPA 管理端同步资源、健康状态和任务执行证据 |
| 违禁词 | 前端共享规则 Store，工作台同步消费 | 服务端版本化词库、审计、配置广播和发送前强校验 |
| 投屏会话 | 浏览器会话标识、`localStorage` 与 `BroadcastChannel` 模拟抢占 | 服务端签发短期投屏会话，同一云电脑原子抢占并主动通知旧会话 |
| 风控与审计 | 状态字段和页面反馈 | 服务端硬拦截、告警闭环、只追加审计和受控证据引用 |

## 2026-07-22 — 资源、企微登录与投屏状态分离

- 云电脑与 RPA 机器人作为独立资源单元维护：一个云电脑对应一个机器人，一个资源最多绑定一个企微号；机器人常驻任务归属 `resourceId`，不依赖企微号显示名。
- 企微账号状态、风控状态、RPA 执行状态、云电脑连接状态和投屏会话状态彼此独立。打开、关闭或抢占投屏不改变企微账号状态，也不自动上线或下线机器人。
- 下线机器人只暂停自动化任务，不关闭云电脑；机器人重新上线必须满足资源可连接、已绑定、企微在线、未封禁、无人工接管和无硬性风控拦截。
- 同一云电脑仅允许一个活跃浏览器投屏会话；新会话挤出旧会话。该规则只约束投屏画面，不创建、删除或替换企微号配置。

## 2026-07-22 — 控制台只承担监控与人工接管

- 企微号生命周期、游戏归属与资源绑定只在 ops-admin 维护；`/control` 左侧仅消费“已配置 + 当前人已授权”的企微号，不提供独立新增、删除或选号登录入口。
- 历史口径曾把撤回放在控制台；现已由顶部一致性决定覆盖：控制台只承担人工接管，消息撤回走独立业务请求。
- 登录、退出或重启企微只改变企微登录状态。操作完成后 RPA 不自动恢复，管理员需回到资源管理明确上线机器人。

## 2026-07-22 — 违禁词与混发结果口径

- 每个游戏最多一条违禁词规则，一条规则维护多个词条；状态只在列表启停，新建规则默认启用。
- 规则仅校验客服出站文本，不校验附件、图片、视频或玩家入站消息。文本命中后使用统一发送失败样式，不提供自动替换或绕过入口。
- 文本和多个附件可以一次提交为一个消息发送批次，并按单条内容记录状态；硬风控命中的文本不入队，合规附件独立执行。

## 2026-07-20 — 领域职责与 V1 范围

- ops-admin 负责企微号配置、唯一游戏归属、云电脑与 RPA 资源绑定、违禁词和运行风控；permission 只负责客服账号与既有企微号之间的访问授权。
- 游戏目录是只读主数据，V1 不建设游戏 CRUD。企微号 `accountId` 是跨域稳定主键，显示名不可作为权限或审计主键。
- 历史方案曾允许删除企微号配置；现已覆盖为仅启用 / 停用，配置、历史、授权引用与资源绑定均保留。
- IP 一致性、单号单桌面、频率控制、封禁处理和应急回退是硬性风控，系统管理员也不能从前端跳过。

## 继续跟进的外部对齐项

- 阿里云云电脑与 RPA 管理端的资源清单、状态同步、任务执行和失败证据接口。
- 无影云投屏协议、会话签发、抢占通知、超时和关闭语义。
- 每企微号发送频率、连续失败率和告警时间窗口的初始策略值；设计采用服务端可配置策略，不在前端硬编码。
- 敏感字段清单、录屏 / 原始日志访问、审计导出和组织级保留策略。当前设计默认审计保留 180 天、证据不可直接下载，最终以安全与合规结论为准。
- V1 试运行规模：游戏、企微号、云电脑、机器人和并发投屏数量。
