# 项目品牌规范

> 默认保存为 `project/ui-brand.md`。
> 这是一份项目级默认设计规范;所有领域的 `design.md` 默认继承这里。只有领域特有的例外才写回对应领域文档。

## 使用说明

- 这份文档只维护项目级、长期稳定的视觉与品牌规则。
- 领域业务规则、页面流程和功能细节写到 `project/domains/<domain>/design.md`。
- 如果只是某个页面临时偏离默认规范,在对应 `design.md` 中写"设计例外说明",不要回写这里。
- 如果某项尚未确定,可暂时沿用 starter 默认值,但应优先在本文件补齐,不要分散到多个领域文档。

## 品牌基础

- 品牌名称:**ChatFlow**(VIP 私域运营客服工作台)
- 品牌关键词:专业 / 高效 / 克制 / 工具感 / 企微生态延续
- 目标气质:B 端会话工具(对标七鱼、腾讯企业点、飞书客户端的工具区);IM + 后台混合,而非纯管理后台
- 适用范围:ChatFlow 所有领域的 Web 前端(V1 仅桌面端,尺寸基线 1440×900)

## 色彩规范

### 品牌色

- 主色:`#07C160`(延续企业微信 / 微信生态的经典绿,降低客服学习成本)
- 主色浅阶(hover / 选中背景 / 徽章底):`#E8F8EE`
- 主色中阶(主色禁用、弱化填充):`#95E1B5`
- 主色深阶(active / 强调文字 / 按钮按下):`#06A052`
- 辅助色(次要按钮、图标):`#595959`(中性深灰,不抢主色)
- 强调色(关键操作链接、高亮徽章):`#1677FF`(信息蓝,和主色有区分度,避免同色歧义)

### 语义色

- 成功:`#52C41A`(接近主色但稍偏黄绿,避免与"主操作"按钮混淆)
- 警告:`#FAAD14`(橙,用于"发送中""企微掉线"等注意级别)
- 错误:`#FF4D4F`(红,发送失败红标、封号告警的强烈信号)
- 信息:`#1677FF`(与强调色一致,普通信息 toast)

### 中性色

- 主文本:`#262626`(正文、表格、输入内容)
- 次文本:`#595959`(说明文案、次级标签、时间戳)
- 辅助文本:`#8C8C8C`(占位符、禁用、辅助说明)
- 一级边框:`#D9D9D9`(卡片外框、输入框正常态)
- 二级边框:`#F0F0F0`(表格行分割、区块轻分隔)
- 页面底色:`#F5F5F5`(工作台外层背景,区分卡片)
- 卡片底色:`#FFFFFF`(会话列表、消息流、表单等主工作区)

## 字体与排版

- 标题字体:`PingFang SC, Microsoft YaHei, system-ui, -apple-system, sans-serif`
- 正文字体:同标题(B 端不分字族,保持一致)
- 代码或数字字体:`SF Mono, Menlo, Monaco, Consolas, 'Courier New', monospace`(企微号 ID、手机号、玩家 UID 等数字字段用等宽对齐)
- 主标题字号:20px(页面级标题)
- 次级标题字号:16px(区块标题、抽屉标题)
- 正文字号:**13px**(紧凑密度核心,比 Ant Design 默认 14 压 1px,同屏多容纳 ~8% 信息)
- 辅助字号:12px(时间戳、说明、徽章计数)
- 默认字重:400(normal)
- 强调字重:500(semi-bold,**不用** 600+,B 端避免视觉过重)
- 默认行高:1.5(配合紧凑字号,消息气泡 1.55)

## 形状与密度

- 默认圆角:**6px**(按钮、输入框、卡片、抽屉)
- 消息气泡圆角:**8px**(比全局大 2,在 IM 场景视觉更柔和) — 领域内例外,写在 chat-workbench `design.md`
- 图像/头像圆角:**50%**(圆形头像)
- 轻量阴影:`0 1px 2px rgba(0, 0, 0, 0.06)`(卡片默认)
- 强调阴影:`0 4px 12px rgba(0, 0, 0, 0.12)`(弹窗、抽屉、下拉)
- 页面内容区密度:**紧凑**
- 组件内部间距基线:**4px**(间距序列:4 / 8 / 12 / 16 / 24)
- 页面区块间距:16px 为主,跨区块 24px
- 控件高度:默认 32px(输入框、按钮);紧凑表单 28px;大按钮 36px
- 列表行高:表格 40px;会话列表 56px(含头像 + 两行信息);消息气泡最小 32px

## 组件气质与默认做法

- **导航与菜单**:顶栏 48px + 左侧垂直主导航(40-48px 宽的图标条 + 展开二级菜单);不做顶部横导航;当前项用主色左边框 + 主色浅阶背景表达。
- **列表与表格**:默认紧凑行高 + 无竖向网格线 + 表头浅灰底;批量操作放表头上方,单行操作放行尾;空态 + 加载 + 错误态统一使用 Ant Design `Empty` / `Spin` / `Alert`。
- **表单**:label 左对齐 + 控件宽度统一;必填星号用错误色;校验错误实时提示(blur 后),不等提交才报错。
- **弹窗 / 抽屉**:**优先用抽屉**(Drawer,从右侧滑入,宽度 480-640px 视内容),尤其是玩家档案、发送失败详情、登录引导等会话内联场景;纯二次确认可用 Modal。
- **标签页**:默认下划线 Tab(`type="line"`),不使用胶囊 Tab(`type="card"` 除非是容器级切换);Tab 高度 40px。
- **卡片**:**弱化卡片**,以 1px 边框 + 轻量阴影为主,避免大面积白底 + 强阴影;卡片内容区 padding 12-16px。
- **数据展示**:图标 16px 为主;企微号、玩家 ID、UID 等数字用等宽字体 + 次文本色;状态用小徽章(高 20px)+ 语义色点。

## 页面氛围

- 背景基调:外层 `#F5F5F5` + 工作区 `#FFFFFF`,形成"桌面 + 纸张"的层级感。
- 视觉层级:结构 > 色彩。用**边框、间距、字重**表达层级,少用色块填充。
- 强调信息表达:主色小面积(徽章、计数、左边框、按钮填充),不要大色块。
- 弱化信息表达:灰度降级 + 折叠 + 次级字号;非主路径操作下沉到"更多"菜单。
- 状态表达:绿=正常 / 橙=注意 / 红=失败或封禁 / 灰=离线或禁用,整项目统一,不反用。

## 禁用样式

- 不要使用的颜色:
  - 品牌绿之外的鲜艳绿(避免语义混淆)
  - 高饱和紫、粉、霓虹色(和 B 端克制气质冲突)
  - 超过 2 色的渐变
- 不要使用的布局倾向:
  - 大面积卡通插画、海报风 hero 区
  - 毛玻璃(backdrop-blur)作为整块背景
  - 全宽居中的"落地页"式排版
- 不要使用的组件风格:
  - 胶囊按钮(`border-radius: 999px`)作为主要按钮样式
  - 立体阴影 / Neumorphism / 3D 凸起按钮
  - 动效炫技(大幅 bounce、长时 transition、视差滚动)
- 不要出现的品牌冲突:
  - 大幅 LOGO 占据主工作区
  - 引用游戏美术素材作为装饰(避免和 IM 工具气质冲突)
  - 模仿企微客户端像素稿(要延续生态感,但视觉是独立品牌)

## 例外原则

- 允许在领域 `design.md` 中声明例外的场景:
  - IM 会话气泡、头像、消息状态等与管理后台风格必然不同的 IM 组件
  - 合规强告警(封号、删好友、发送失败)需要比全局规范更显眼的视觉表达
  - 数据看板或图表页(未来 monitor 领域启用时)的配色与密度会与主工作台不同
- 例外说明应包含:
  - 例外点是什么(哪个字段、颜色、间距、控件)
  - 为什么必须例外(业务诉求 / 技术约束 / 可用性理由)
  - 影响范围(仅本页 / 整个领域 / 跨领域)
  - 是否需要同步到实现(CSS 变量、组件 props)
