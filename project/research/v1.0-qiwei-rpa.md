## 前期规划草稿

### 业务目标

- 为 VIP 玩家运营提供**自建后台**,把多个企微账号下的消息**聚合管理 + 统一回复**
- 解决现状:账号分布不均、运营工作分配不合理、玩家归属感低、排班需要 1+1 双开号
- 核心指标方向:运营效率(接待时长、绩效可量化)、玩家体验(响应时长、品牌口碑),间接影响 VIP 留存

### 核心约束

- 企微官方**只开放消息回调 API,不开放消息发送 API**;单纯走官方能力无法满足"从自建后台向玩家主动发消息"
- 业界现成方案(七鱼、乐变)选型都是 **RPA** 模拟人工操作,无绝对安全方案,但 RPA + 云桌面是当前较稳妥的选择
- 风控红线:封号、敏感信息泄露、IP 漂移、操作频率异常

### 技术路线选项

1. **完全自研 RPA**:开发成本高、不确定性大,业务契合度反而低 — 不采纳
2. **部分自研 + 三方 RPA 通信(阿里云 RPA)**:业务相关自研,通信层接三方 — **本项目采纳方向**
3. **接入三方(百胜软件等)整套 SaaS**:后台能力现成,但自主可控度低、定制空间小 — 备选

### 阿里云 RPA 接入方式对比

| 接入方式    | 适用阶段  | 灵活性 | 说明                             |
| ------- | ----- | --- | ------------------------------ |
| 可视化组件编辑 | 原型/验证 | 低   | 上手门槛低,PM 可用于需求流程挖掘,执行效率有限 |
| 代码开发(SDK) | 生产推荐 | 高   | 能处理复杂操作、性能更优 — **本项目推荐主路径** 🌟 |
| 接入三方后台 / API | 快速起步 | 中   | 百胜软件:一机多开、群发、群管理、任务管理等能力现成;发送速度 ≈ 1.6s/条 |

### 功能 / 能力规划雏形

- 企微账号池管理(账号登录态、云桌面绑定关系、在线/离线/未读状态可视化)
- 消息聚合收件箱(跨账号消息统一视图)
- 消息发送(单发、群发、带媒体)
- 运营分配与工作量统计(人均维护账号、消息量、绩效可量化)
- 敏感信息脱敏展示(玩家账号、联系方式、充值金额)
- 操作/异常监控(掉线、封禁、发送失败、速度监控、日志与视频回放)
- 应急预案通道(回退企微官方后台、预留云服务器资源)

- 需要补 `project/domains/<domain>/delivery/prd.md`:项目尚未建任何业务领域目录,**首个领域(建议名:`vip-im` / `message-hub` 等)需要先建立**,再从本文件迁入背景、约束、风控规则
