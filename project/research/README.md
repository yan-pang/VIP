# 项目级调研

> 跨领域、针对整个项目的调研文档汇总。  
> 业务领域内的细节不走这里,直接写进对应领域 `design.md`。

## 组织方式

- 按迭代版本命名文件:`v1.0-market-scan.md`、`v1.1-user-interview.md`
- 同一版本允许多份(不同调研主题分文件)
- 单份文件内容参考 `product-design-kit/design/research-doc-template.md`

## 职责边界

- 不沉淀完整会议纪要或研发交付正文,只保留来源和对 PRD / 测试有用的提炼结果
- 影响 demo 设计的结论,回写到对应领域 `design.md`
- 需要对外交付的参考点,提炼进 `project/delivery/v1.x/release-prd.md` 或对应领域 PRD
