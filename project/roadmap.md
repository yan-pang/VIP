# ChatFlow 产品路线图

> 项目代号 **ChatFlow**:面向游戏 VIP 玩家的私域运营客服后台。
> 架构路线:业务自研后台 + 阿里云 RPA + 无影云,绕过企微"无消息发送 API"限制。
> 本文件由总体规划固化,是版本间优先级的权威表述;发版产物放在 `project/delivery/v1.x/`。

## 系统定位

- 与现有 SCRM 并行互通的**全新产品**,长期目标逐步承接 SCRM 能力。
- 当前阶段按独立后台设计;权限、客服账号、企微号绑定全部由 ChatFlow 自身领域承担,V1 原型不依赖外部身份/权限平台。
- 前置调研见 [project/research/v1.0-qiwei-rpa.md](research/v1.0-qiwei-rpa.md)。

## 版本 Roadmap

### v1.0 — 客服工作台 MVP(本轮重点)

少量企微号 + 少量客服的闭环演示,等同"能交付给运营试用的最小后台"。
**取舍**:V1 不做客服状态机、接待上限、快捷回复,客服登录即在岗、下线即走人,消息纯手工输入。

共 **29 条功能**(17 P0 + 4 P1 + 8 闭环补位),清单见本文件「V1 条目清单」。

### v1.1 — 运营效率增强

- 客服状态机(在线/离线/小休)、接待上限、状态操作日志
- 智能路由(均衡分配为主)、会话自动转接联动
- 会话挂起/小休、快捷回复管理(个人 + 公共)、消息撤回入口完善
- 玩家游戏信息联动(FPID、区服、充值等)
- 好友验证后台可视化(只读,操作仍由运营手机端/无影云完成)

### v1.2 — 内容合规与质量

- 链接白名单、内容质检(对接 SCRM)、自定义字段完整化
- 综合搜索体验增强、标签组管理

### v1.3 — 运营洞察

- 客服 / 客服组 / 会话监控、报表(客服、响应度、满意度、玩家)
- 操作日志完整化、排队通知

### v1.4+ — 自动化与 AI

- FAQ 知识库 / Quick Reply 模板化 / AI 值守 / 评价管理 / 智能场控
- 群发与私域营销联动
- 承接 SCRM 历史能力

> 后续版本的条目和时间由 `/deliver` 基于领域 PRD 的 diff 组装,不在本文件预先锁定。

## 待对齐的外部依赖

这些不阻塞 V1 原型,但落到真实实现前必须有答案:

- **企微会话存档 API 开通与接入**:R021 前置条件、数据拉取频率
- **无影云 API 接入契约**:R018/R020 的配额与超时;`/control` 视频流投屏协议
- **云桌面手动撤回入口**:R045 的具体跳转链接形式(由无影云或运维方提供)
- **SCRM 数据互通接口**:标签、玩家档案双向同步协议(R014/R056)
- **敏感信息脱敏字段清单**:前期调研已提出,未明确字段
- **V1 试运行规模**:实际投入企微号数量与先试游戏

这些会在 `project/research/v1.0-qiwei-rpa.md` 的"待确认问题"里扩充追踪,并分散进各领域 `design.md` 的"待澄清"章节。
