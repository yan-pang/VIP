# 项目级研发设计

> 跨领域、按迭代版本增量的研发技术设计文档汇总。  
> 由研发同学主维护,PM 可参考。

## 组织方式

- 按迭代版本命名文件:`v1.0-architecture.md`、`v1.1-db-redesign.md`
- 同一版本允许多份(不同技术主题分文件)
- 常见内容类型:整体架构、服务划分、数据库设计、接口规范、基础设施、迁移方案

## 职责边界

- 研发设计默认不回写领域 `design.md`;只有会改变 demo 表现(页面 / 交互 / Mock)时才同步
- 需要对外交付的技术约束、接口、依赖、异常路径,提炼进对应领域 PRD 或版本级 `test-plan.md`
- 这里不放单领域的技术细节;那种内容留在对应领域 `design.md` 的"设计例外"或附注里
